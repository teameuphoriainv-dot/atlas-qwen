## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can run it and see something real.
2. **Infrastructure before features.** Phase 0 sets up everything so feature work is fast and clean.
3. **Magic moment first.** The natural-language-order → confirm → write-back loop works by the end of Phase 2. Everything else builds on it.
4. **Trust is built in, not bolted on.** The PHI-isolation boundary and its test land in Phase 1, before any model call — never after.
5. **Test as you go.** Each task includes verification. The PHI-isolation test is part of `npm test` from Phase 1 onward.
6. **Progressive enhancement.** Simplest working version first (hardcoded demo patient, text input), then polish (picker, streaming, voice).
7. **Hackathon reality:** when the clock is short, do Phases 0–2 to nail the demo, then cherry-pick Phase 3 polish that most improves the live run. (Optional) push each phase as a PR for a review agent like [CodeRabbit](https://coderabbit.ai) if time allows — it catches issues the coding agent won't flag.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists its Reference sections — read only those parts of docs/prd.md and docs/product-vision.md, not the whole documents.
3. **Let it work:** The agent finds the first unchecked task, implements it, and marks it `- [x]`.
4. **One session = one phase (ideally):** Best continuity. If you stop, resume from the first unchecked task.
5. **(Optional) Push a PR for review:** At a hackathon this is optional, but if you have time, push each phase as a PR and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) check it before continuing — it catches bugs and security issues the coding agent may miss.
6. **Need more context?** If a task references a section not in the phase's Reference sections, read just that section on demand.

### Session Tips
- **Don't read everything:** Use each phase's Reference sections to stay focused and save context.
- **Don't skip tasks:** They're ordered to resolve dependencies without jumping around.
- **Verify after each phase:** Run the app and confirm the demoable outcome before moving on.
- **Protect the magic moment:** Phase 2 is the demo. If the clock is short, get Phases 0–2 perfect before touching Phase 3.
- **Update the status line:** After completing tasks, update `**Status:** X/41 tasks complete` and `**Current Phase:** Phase N`.
